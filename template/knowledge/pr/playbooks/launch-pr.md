# Launch PR Playbook

## Timeline

| Day | Activity | Owner |
|-----|----------|-------|
| D-14 | Finalize messaging framework and key messages | pr-strategist |
| D-10 | Draft press release | pr-writer |
| D-10 | Begin press kit assembly | pr-writer |
| D-7 | Build targeted media list (10-20 journalists) | pr-outreach |
| D-7 | Finalize press release (client approval) | pr-coordinator |
| D-5 | Draft personalized pitches for Tier 1 | pr-outreach |
| D-3 | Send embargoed pitches to Tier 1 journalists | pr-outreach |
| D-1 | Prep social amplification posts | pr-coordinator |
| D-0 | Public release: distribute to Tier 2-3, post social | pr-coordinator |
| D+1 | Follow up with non-responsive Tier 1 | pr-outreach |
| D+3 | First coverage check | pr-monitor |
| D+5 | Follow up with Tier 2-3 | pr-outreach |
| D+7 | Coverage report | pr-monitor |
| D+14 | Final coverage report + lessons learned | pr-monitor |

## Embargo Rules
- Tier 1 outlets get 3-day advance access
- Embargo = journalist agrees not to publish until release date
- Always confirm embargo acceptance in writing (email reply)
- If embargo breaks, immediately release to all media
- Never embargo more than 3-5 journalists per launch

## Media Tiering for Launches
- **Tier 1:** National tech press (TechCrunch, Wired, The Verge, VentureBeat)
- **Tier 2:** Industry publications (SaaS-specific, fintech, AI trade press)
- **Tier 3:** Niche blogs, newsletters, podcasts

## Deliverables
1. Messaging framework
2. Press release (final)
3. Press kit (complete)
4. Media list (tiered)
5. Personalized pitches (per journalist)
6. Social amplification posts
7. Coverage report
