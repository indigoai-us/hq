Steve Yegge's taxonomy of how developers progress through AI-assisted coding, from the Gas Town article. Positioned as a prerequisite assessment — Gas Town requires Stage 6-7 minimum.

## The Eight Stages

1. **Zero/Near-Zero AI**: Code completions, occasional chat questions.
2. **IDE agent, permissions on**: Narrow sidebar agent asks permission to run tools.
3. **IDE agent, YOLO mode**: Trust increases, permissions disabled, agent scope widens.
4. **IDE, wide agent**: Agent fills the screen. Code is just for diffs.
5. **CLI, single agent, YOLO**: Diffs scroll by. May or may not review them.
6. **CLI, multi-agent, YOLO**: 3-5 parallel instances. Very fast throughput.
7. **10+ agents, hand-managed**: Pushing limits of manual coordination.
8. **Building own orchestrator**: On the frontier, automating workflow.

## Implication

Gas Town targets Stage 7-8 developers who have already committed to vibe coding — treating work as throughput-focused, tolerating some chaos and redundancy in exchange for massive parallelism. The Overseer role is essentially a Product Manager: designing features, filing plans, slinging work to agents.
